00. 시작하기

처음 접속하시는 운영자를 위한 안내입니다.

1. 접속 방법

주소

  • 운영 서버: https://halle-bf789.web.app/admin/login

초기 로그인

항목값
아이디admin
비밀번호admin

🔐 첫 로그인 후 반드시 비밀번호를 변경해 주세요.

변경 방법: 좌측 메뉴 → 시스템 → 설정 → 운영자 계정 → 비밀번호 변경 버튼

자동 로그인

로그인 화면에서 자동로그인 체크박스를 선택하면 다음 접속 시 로그인 없이 바로 들어갈 수 있습니다.

로그아웃

화면 우측 상단 로그아웃 버튼

2. 화면 구성

좌측 사이드바

모든 메뉴가 카테고리별로 정리되어 있습니다.

  • 운영 — 회원, 팀, 경기, 매칭, 구장 관리
  • 커뮤니티 — 게시판, 채팅 관리
  • 콘텐츠 — 광고, 알림, 앱 업데이트
  • 시스템 — 운영자 계정, 약관/정책

화살표(▼)가 있는 메뉴는 클릭하면 하위 메뉴가 펼쳐집니다.

상단바

  • 좌측: 현재 위치(빵부스러기 경로)와 접속 시각
  • 우측: 로그인된 운영자 이름 + 로그아웃 버튼

3. 대시보드

로그인하면 바로 보이는 화면입니다.

주요 통계

  • 회원 수 — 전체 가입 회원 수
  • 팀 수 — 등록된 팀 수
  • 활성 매칭 — 진행 중인 매칭 수
  • 오늘 가입자 — 오늘 새로 가입한 회원 수

숫자는 자동으로 갱신됩니다. 새로고침 시 최신값이 반영됩니다.

빠른 이동

대시보드의 카드를 클릭하면 해당 관리 페이지로 바로 이동할 수 있습니다.

4. 처음 운영자가 해야 할 일 (체크리스트)

배포 직후 한 번만 진행하시면 됩니다.

  • [ ] admin 계정 비밀번호 변경

    • - 시스템 → 운영자 계정 → admin 행의 비밀번호 변경 클릭

  • [ ] 추가 운영자 등록 (필요 시)

    • - 시스템 → 운영자 계정 → + 운영자 추가

  • [ ] 약관·정책 검토

    • - 시스템 → 약관/정책 → 개인정보 처리지침 / 이용약관 본문 확인

    - 회사 정책에 맞게 수정 후 저장

  • [ ] 홈 배너 1개 이상 등록

    • - 콘텐츠 → 홈 배너 → + 새 배너 등록

  • [ ] 추천 구장 등록

    • - 운영 → 구장 관리 → + 새 구장 등록

01. 회원 관리

메뉴: 운영 → 회원 관리

회원 목록 조회, 신고 처리, 차단 관리를 할 수 있습니다.

회원 목록

경로: 회원 관리 → 회원 목록

볼 수 있는 정보

  • 닉네임 / 이름 / 전화번호 / 지역 / 가입일
  • 활동 통계 (참여 매칭 수 등)

검색·필터

  • 검색어 (닉네임/이름/전화번호)
  • 가입일 기간
  • 25개씩 페이지네이션

할 수 있는 작업

  • 상세 보기 — 행 클릭 시 회원의 상세 프로필 확인
  • 차단 — 부적절 회원 즉시 차단

회원 신고

경로: 회원 관리 → 회원 신고

다른 사용자가 회원을 신고하면 여기에 쌓입니다.

처리 방법

버튼동작
차단신고 사유를 검토 후 회원을 차단 (신고는 자동으로 종결)
기각신고 내용이 부적절하거나 사실무근일 때 (회원은 그대로)

탭 필터

  • 대기 — 아직 처리하지 않은 신고
  • 차단됨 — 차단으로 처리된 신고
  • 기각 — 기각된 신고
  • 전체 — 모든 상태

차단 진행 시

  • 차단 사유 입력 모달이 열림 → 신고 내용을 참고해서 사유 작성
  • 차단하기 버튼 클릭 → 회원이 차단되고 자동으로 차단 회원 페이지로 이동

차단 회원

경로: 회원 관리 → 차단 회원

차단 처리된 회원을 모아서 볼 수 있습니다.

볼 수 있는 정보

  • 닉네임 / 전화번호 / 지역
  • 차단 사유 / 차단일 / 처리자

할 수 있는 작업

  • 차단 해제 — 정당한 사유 확인 후 다시 활성화
  • 상세 보기 — 차단된 회원의 프로필 확인

자주 묻는 질문

Q. 차단된 회원은 앱을 사용할 수 있나요?

  • 차단된 회원은 로그인은 되지만 모든 기능이 제한됩니다. (자동 안내 화면 표시)

Q. 신고를 잘못 차단했어요. 되돌릴 수 있나요?

  • 차단 회원 페이지에서 해당 회원을 찾아 차단 해제 가능합니다.

Q. 회원이 직접 탈퇴할 수 있나요?

  • 네. 마이페이지에서 회원 탈퇴가 가능합니다. 단, 탈퇴 후에는 복구되지 않습니다.

02. 팀 관리

메뉴: 운영 → 팀 관리

팀 목록 조회, 신고 처리, 차단 관리를 할 수 있습니다.

팀 목록

경로: 팀 관리 → 팀 목록

볼 수 있는 정보

  • 팀 이름 / 팀장 / 지역 / 멤버 수 / 생성일
  • 팀 활동 통계 (경기 수, 승률 등)

검색·필터

  • 검색어 (팀명/팀장명)
  • 지역 필터
  • 25개씩 페이지네이션

할 수 있는 작업

  • 상세 보기 — 행 클릭 시 팀 정보 미리보기
  • 차단 — 부적절 팀 즉시 차단

팀 신고

경로: 팀 관리 → 팀 신고

사용자가 팀을 신고하면 여기에 쌓입니다.

처리 방법

버튼동작
차단신고 사유 검토 후 팀 차단
기각신고가 부적절하거나 사실무근일 때

탭 필터

  • 대기 / 차단됨 / 기각 / 전체

차단 팀

경로: 팀 관리 → 차단 팀

차단된 팀을 모아서 봅니다.

할 수 있는 작업

  • 차단 해제 — 다시 활성화
  • 상세 보기 — 차단된 팀 정보 확인

자주 묻는 질문

Q. 팀을 차단하면 팀 멤버들도 모두 차단되나요?

  • 아닙니다. 팀만 차단되며, 팀 멤버는 개인 계정으로 정상 활동 가능합니다.

Q. 팀 차단 시 진행 중인 매칭은 어떻게 되나요?

  • 차단된 팀의 매칭은 자동으로 취소됩니다.

Q. 팀이 해체되면 어떻게 되나요?

  • 팀장이 직접 해체하면 멤버는 자동으로 분리됩니다. 어드민 차단과는 별개입니다.

03. 경기 관리

메뉴: 운영 → 경기 관리

KBL/NBA 등 외부 경기 일정을 조회할 수 있습니다.

예정된 경기

경로: 경기 관리 → 예정된 경기

앞으로 열릴 경기 일정을 확인합니다.

볼 수 있는 정보

  • 리그 / 경기일시 / 홈팀 ↔ 어웨이팀 / 장소
  • 자동 크롤링된 데이터 (수동 입력 불필요)

검색·필터

  • 리그별 (KBL / NBA 등)
  • 날짜 기간

지난 경기

경로: 경기 관리 → 지난 경기

이미 끝난 경기 결과를 확인합니다.

볼 수 있는 정보

  • 리그 / 경기일시 / 홈팀 점수 ↔ 어웨이팀 점수 / 결과
  • 자동 크롤링된 데이터

동작 방식

  • 경기 데이터는 외부 사이트(네이버 스포츠 등) 에서 자동으로 수집됩니다.
  • 매일 새벽과 15분마다 자동 업데이트되므로 운영자가 따로 등록할 필요가 없습니다.

자주 묻는 질문

Q. 경기 데이터가 안 보여요.

  • 자동 크롤링 일정에 따라 약간의 지연이 있을 수 있습니다.
  • 1시간 이상 안 보이면 개발자에게 문의해 주세요.

Q. 경기 결과가 잘못됐어요.

  • 외부 사이트의 데이터를 가져오므로, 원본이 잘못된 경우 자동으로 수정됩니다.
  • 수동 수정이 필요하면 개발자에게 문의해 주세요.

04. 매칭 관리

메뉴: 운영 → 매칭 관리

회원/팀 간 매칭 현황을 모니터링할 수 있습니다.

매칭 목록

경로: 매칭 관리 → 매칭 목록

볼 수 있는 정보

  • 매칭 ID / 호스트 팀 / 상대 팀 / 매칭일시 / 장소 / 상태

매칭 상태

  • 모집중 — 호스트 팀이 상대를 찾고 있음
  • 확정 — 매칭이 성사됨
  • 진행중 — 경기 진행 중
  • 종료 — 경기가 끝남
  • 취소 — 매칭이 취소됨

검색·필터

  • 상태 필터
  • 매칭일 기간
  • 25개씩 페이지네이션

할 수 있는 작업

  • 상세 보기 — 매칭의 상세 정보 (라인업, 채팅, 결과 등)

분쟁/신고 (준비중)

경로: 매칭 관리 → 분쟁/신고

🚧 이 기능은 현재 준비중입니다.

매칭 진행 중 발생한 분쟁이나 노쇼 신고를 처리할 예정입니다.

자주 묻는 질문

Q. 매칭을 어드민이 강제 취소할 수 있나요?

  • 현재는 자동 모니터링 위주이며, 강제 취소 기능은 추가 개발 예정입니다.

Q. 매칭 결과를 수정할 수 있나요?

  • 매칭 결과는 양 팀이 동의해야 확정됩니다. 운영자 임의 수정 기능은 향후 분쟁 처리 기능에 포함될 예정입니다.

05. 구장 관리

메뉴: 운영 → 구장 관리

할래말래에서 추천하는 구장(체육관/실외 코트)을 등록·수정·삭제합니다.

사용자는 매칭 시 등록된 구장 중에서 선택할 수 있습니다.

새 구장 등록

  1. 우측 상단 + 새 구장 등록 버튼 클릭 → 모달 오픈
  2. 정보 입력

    3. - 구장명 (필수): 예) 고려대 화정체육관

    - 주소 (필수): 예) 서울 성북구 고려대로 1

    - 상세 주소 (선택): 예) 지하 1층 농구코트

    - 위도 / 경도 (선택): 지도 표시용

    - 종류: 실내(체육관) / 실외(코트) / 기타

    - 이용 비용: 무료 / 유료 / 협의

    - 메모: 이용 안내, 주차 정보 등

    - 표시 순서: 작을수록 앞에 노출

    - 사용자에게 노출 (활성) 체크박스

  3. 좌측 이미지 영역 클릭 → 구장 이미지 업로드 (권장 1200×800)
  4. 구장 등록 버튼 클릭 → 등록 완료

구장 수정

  • 목록에서 해당 구장의 수정 버튼 클릭 → 같은 모달이 열림
  • 수정 후 수정 저장 버튼

구장 삭제

  • 목록에서 삭제 버튼 클릭 → 확인 후 영구 삭제
  • 삭제하면 사용자가 해당 구장을 더 이상 선택할 수 없습니다.

활성/비활성

  • 사용자에게 노출 (활성) 체크박스로 제어
  • 비활성 상태인 구장은 사용자에게 노출되지 않지만, 어드민 목록에는 보입니다.
  • 일시적으로 안 보이게 하고 싶을 때 활용

샘플 구장 자동 등록

운영 초기 테스트용으로 샘플 2개 자동 등록 버튼이 있습니다.

  • 동서울대 체육관, 한국외대 미네르바콤플렉스 체육관
  • 이미지·주소·좌표 포함
  • 한 번만 실행하세요. (중복 등록 방지)

자주 묻는 질문

Q. 위도/경도는 어떻게 입력하나요?

  • 카카오맵이나 구글맵에서 구장 위치를 검색 → 위도/경도 복사
  • 입력하지 않아도 등록은 가능하지만, 지도 표시에 제한이 있습니다.

Q. 이미지를 안 올리면 어떻게 되나요?

  • 등록은 가능하지만, 사용자 화면에 기본 이미지가 표시됩니다.

Q. 한 구장에 여러 사진을 올릴 수 있나요?

  • 현재는 한 장만 가능합니다.

06. 게시판

메뉴: 커뮤니티 → 게시판

사용자가 작성한 게시글을 관리하고, 신고된 게시글을 처리합니다.

게시글 (커뮤니티 → 게시판 → 게시글)

볼 수 있는 정보

  • 제목 / 작성자 / 댓글 수 / 좋아요 / 조회수 / 작성일 / 상태

탭 필터

  • 전체 — 모든 게시글
  • 공개 — 사용자에게 보이는 게시글만
  • 숨김 — 어드민이 숨김 처리한 게시글
  • 공지 — 공지로 고정된 게시글만

검색·필터

  • 기간 (작성일 기준)
  • 검색어 (제목 또는 작성자명)
  • 25개씩 페이지네이션

할 수 있는 작업

버튼동작
공지게시글을 공지로 고정 → 사용자 목록 최상단에 노출
공지해제공지 고정 해제
숨김사용자 화면에서 안 보이게 처리 (해제 가능)
숨김해제다시 노출
삭제게시글·댓글·좋아요 모두 영구 삭제 (복구 불가)
제목 클릭상세 페이지로 이동

상세 페이지

  • 본문 / 첨부 이미지 / 댓글 목록 확인
  • 댓글 개별 삭제 가능
  • 상단에서 공지 토글 / 숨김 토글 / 게시글 삭제 가능

💡 공지 / 숨김 / 삭제 선택 기준

- 일시적인 부적절한 글: 숨김 권장 (해제 가능)

- 명백한 위반·스팸: 삭제 권장

- 운영 공지(점검 안내 등): 공지로 고정

신고글 (커뮤니티 → 게시판 → 신고글)

사용자가 게시글을 신고하면 여기에 쌓입니다.

볼 수 있는 정보

  • 신고 대상 게시글 제목 (클릭 → 게시글 상세로 이동)
  • 게시글 작성자
  • 신고자 닉네임 + UID
  • 신고 사유
  • 처리 상태 / 신고일

탭 필터

  • 대기 — 처리 대기중 (이게 보통 가장 먼저 봐야 할 탭)
  • 처리됨 — 숨김 또는 삭제로 처리한 신고
  • 기각 — 사실무근으로 기각한 신고
  • 전체

처리 방법 (대기 상태)

버튼동작
숨김게시글을 숨김 처리하고 신고를 종결
삭제게시글을 영구 삭제하고 신고를 종결
기각게시글은 그대로 두고 신고만 기각

자주 묻는 질문

Q. 사용자는 어떻게 신고하나요?

  • 게시글 상세 화면에서 🚩 게시글 신고하기 버튼을 클릭하고 사유를 입력합니다.
  • 본인이 작성한 글은 신고할 수 없습니다.

Q. 같은 게시글이 여러 번 신고됐어요.

  • 각 신고는 개별 항목으로 표시됩니다. 한 번 처리(숨김/삭제)하면 게시글 자체가 처리되지만, 다른 신고는 별도로 처리해야 합니다.

Q. 신고를 잘못 기각했어요.

  • 기각 탭에서 해당 신고를 다시 확인할 수 있습니다. 다시 처리하려면 개발자에게 문의하세요.

Q. 숨김 처리한 게시글은 작성자에게 알림이 가나요?

  • 별도 알림은 가지 않습니다. 작성자가 자기 글이 안 보이는 걸 보고 인지하게 됩니다.

07. 채팅 관리

메뉴: 커뮤니티 → 채팅 관리

사용자 간 1:1 채팅방을 모니터링하고 관리합니다.

채팅방 목록

볼 수 있는 정보

  • 참여자 (닉네임 + UID 두 명)
  • 마지막 메시지 / 마지막 메시지 시각
  • 채팅방 생성일
  • 상태(활성 / 잠금)

탭 필터

  • 전체 / 활성 / 잠금

검색·필터

  • 기간 (생성일 기준)
  • 검색어 (참여자 닉네임 / UID / 마지막 메시지 내용)
  • 25개씩 페이지네이션

할 수 있는 작업

버튼동작
보기채팅방 상세로 이동 → 메시지 일람
잠금사유 입력 후 잠금 → 사용자가 메시지를 보낼 수 없음
잠금해제다시 사용 가능하게 풀어줌
삭제채팅방 + 모든 메시지 영구 삭제

잠금 동작

  • 잠긴 방에서 사용자가 메시지를 보내려고 하면 "관리자가 잠근 채팅방입니다. 메시지를 보낼 수 없습니다." 라는 에러가 뜹니다.
  • 잠금 사유는 어드민 기록용이며 사용자에게 보이지 않습니다.

채팅방 상세

목록의 보기 또는 참여자 이름을 클릭하면 들어갑니다.

볼 수 있는 정보

  • 상단: 참여자 정보 (이름 + 아바타), 방 ID, 유형, 생성일, 잠금 상태/사유
  • 메시지 일람: 전체 메시지를 시간순으로 표시 (텍스트/이미지)

할 수 있는 작업

  • 방 잠금 / 잠금 해제 (상단 우측 버튼)
  • 방 삭제 (상단 우측 버튼)
  • 메시지 개별 삭제 (각 메시지 우측 삭제 버튼)
  • 메시지 이미지 클릭 → 새 탭에서 원본 보기

자주 묻는 질문

Q. 채팅방에 참여해서 메시지를 보낼 수 있나요?

  • 어드민은 모니터링 전용이며, 메시지를 보낼 수는 없습니다.

Q. 잠금된 방의 메시지도 볼 수 있나요?

  • 네, 잠금 상태와 무관하게 모든 메시지를 확인할 수 있습니다. 새 메시지만 차단됩니다.

Q. 채팅방을 삭제하면 사용자에게 알림이 가나요?

  • 별도 알림은 가지 않습니다. 사용자가 채팅 목록에 들어갔을 때 방이 사라진 것을 보게 됩니다.

Q. DM 외에 그룹 채팅도 있나요?

  • 현재는 1:1 DM만 있습니다.

08. 광고 관리

메뉴: 콘텐츠 → 광고 관리

홈 화면에 노출되는 배너와 진입 시 뜨는 이벤트 팝업을 관리합니다.

홈 배너 (콘텐츠 → 광고 관리 → 홈 배너)

홈 상단에 슬라이드로 노출되는 히어로 배너입니다.

새 배너 등록

  1. 우측 상단 + 새 배너 등록 클릭 → 모달 오픈
  2. 정보 입력

    3. - 이미지 (필수): 좌측 영역 클릭 → 이미지 업로드

    - 권장: 1200 × 600 px (가로:세로 = 2:1) / 5MB 이하

    - 자동 1080px로 압축

    - 텍스트가 들어간 이미지보다 여백이 있는 일러스트/사진 추천

    - 제목 (필수): 줄바꿈은 \n 으로 입력 가능

    - 예) 팀 만들고\n오늘 한 판 어때?

    - 설명: 한 줄 설명

    - 이미지 위치: 왼쪽 / 오른쪽 (텍스트와 좌우 배치)

    - 텍스트 정렬: 왼쪽 / 오른쪽

    - 표시 순서: 작을수록 슬라이드 앞쪽

    - 사용자에게 노출 (활성) 체크박스

  3. 배너 등록 버튼 → 등록 완료

배너 수정·삭제·활성/비활성

  • 목록에서 수정 / 삭제 / 활성화/비활성화 버튼 사용

활용 팁

  • 3~5개 배너를 두면 슬라이드가 자연스럽습니다.
  • 시즌별 이벤트, 신규 기능 안내, 이용 가이드 등으로 활용하세요.

이벤트 팝업 (콘텐츠 → 광고 관리 → 이벤트 팝업)

홈 진입 시 화면 중앙에 뜨는 팝업입니다.

새 팝업 등록

  1. + 새 이벤트 팝업 등록 클릭 → 모달 오픈
  2. 정보 입력

    3. - 이미지 (선택): 720 × 900 px (4:5) 권장 / 5MB 이하

    - 이미지 없이 텍스트만으로도 등록 가능

    - 제목 (필수): 예) 신규 가입 이벤트 진행 중!

    - 본문 (선택): 팝업에 표시할 안내 텍스트

    - 버튼 라벨 (선택): 예) 자세히 보기

    - 이동 URL (선택): 외부 링크 또는 /home/banners 같은 내부 경로

    - 시작 일시 (선택): 비우면 항상 노출

    - 종료 일시 (선택): 비우면 항상 노출

    - 표시 순서: 작을수록 우선

    - 사용자에게 노출 (활성) 체크박스

  3. 팝업 등록 클릭 → 등록 완료

동작 방식

  • 사용자가 홈에 진입할 때 활성 팝업이 표시됩니다.
  • 사용자가 "오늘 하루 보지 않기" 를 누르면 그 날 동안 다시 뜨지 않습니다.
  • 시작/종료 일시를 설정하면 해당 기간에만 노출됩니다.

팝업 수정·삭제·활성/비활성

  • 목록에서 수정 / 삭제 / 활성화/비활성화 버튼 사용

자주 묻는 질문

Q. 배너와 팝업의 차이는요?

  • 배너: 홈 상단에 항상 노출되는 슬라이드. 가벼운 안내·홍보용.
  • 팝업: 홈 진입 시 화면 가운데에 뜨는 강조 알림. 중요한 이벤트나 공지용.

Q. 동시에 여러 팝업이 활성화되면 어떻게 되나요?

  • 표시 순서가 작은 것부터 차례로 노출됩니다. 한 번 닫으면 다음 팝업이 뜹니다.

Q. 비활성화하면 즉시 사용자에게서 사라지나요?

  • 네, 새로고침 시 즉시 반영됩니다.

Q. 외부 URL이 아닌 앱 내부 페이지로 보내고 싶어요.

  • 이동 URL에 /home/community 처럼 슬래시(/) 로 시작하는 경로를 넣으면 됩니다.

09. 알림·푸시

메뉴: 콘텐츠 → 알림/푸시

공지사항을 작성하고 푸시 알림을 발송합니다.

공지 작성

경로: 알림/푸시 → 공지 작성

사용자에게 보여줄 공지사항을 등록합니다.

새 공지 등록

  1. + 새 공지 또는 작성 버튼 클릭
  2. 정보 입력

    3. - 제목 (필수)

    - 본문 (필수): 줄바꿈 보존

    - 카테고리 (선택): 일반 / 이벤트 / 점검 등

    - 표시 순서

    - 활성 여부

  3. 등록 버튼

공지 수정·삭제

  • 목록에서 수정 / 삭제 / 활성화/비활성화

사용자에게 노출되는 곳

  • 마이페이지 → 공지사항 메뉴
  • 알림 → 공지 탭

푸시 발송 (준비중)

경로: 알림/푸시 → 푸시 발송

🚧 이 기능은 현재 준비중입니다.

향후 개발 예정 사항:

- 전체 회원 / 특정 회원 / 특정 팀 대상 푸시 발송

- 발송 결과 자동 기록

발송 로그 (준비중)

경로: 알림/푸시 → 발송 로그

🚧 이 기능은 현재 준비중입니다.

향후 개발 예정 사항:

- 푸시·인앱 알림 발송 이력 조회

- 발송 성공/실패 통계

- 실패 사유 확인

자주 묻는 질문

Q. 공지를 등록하면 푸시도 자동으로 가나요?

  • 현재는 별도입니다. 공지는 화면에만 등록되고, 푸시는 별도 발송이 필요합니다.
  • 향후 "공지 등록 시 푸시 발송 옵션"을 추가할 예정입니다.

Q. 사용자가 푸시를 안 받게 설정했어요.

  • 사용자가 마이페이지 → 알림 설정에서 직접 끌 수 있습니다.
  • 끈 사용자에게는 발송되지 않습니다.

Q. 푸시 발송 기능은 언제 추가되나요?

  • 추가 개발 일정은 개발자에게 문의해 주세요.

🚧 이 기능은 현재 준비중입니다.

  • 등록일

권한 종류

  • 최고 관리자 (보라색) — admin 계정. 시스템에서 자동 생성되며 삭제할 수 없습니다.
  • 운영자 (초록색) — 추가로 등록한 일반 운영자. 삭제 가능.

💡 본인 계정 옆에는 (나) 표시가 붙습니다.

운영자 추가

  1. 우측 상단 + 운영자 추가 클릭 → 모달 오픈
  2. 정보 입력

    3. - 아이디: 영문/숫자/_/- 3~20자 (예: manager1)

    - 이름: 표시될 이름 (예: 홍길동)

    - 비밀번호: 4자 이상

  3. 추가 버튼 → 즉시 해당 계정으로 어드민 로그인 가능

추가된 계정은 일반 운영자 권한입니다.

비밀번호 변경

  1. 변경할 운영자 행의 비밀번호 변경 버튼 클릭
  2. 새 비밀번호 입력 (4자 이상)
  3. 변경 버튼

admin 본인 비밀번호도 동일한 방식으로 변경 가능합니다.

운영자 삭제

  1. 삭제할 운영자 행의 삭제 버튼 클릭
  2. 확인 → 즉시 삭제

삭제 불가 케이스

  • admin (최고 관리자) — 시스템 보호로 삭제 불가
  • 현재 로그인된 본인 계정 — 실수 방지로 삭제 불가

    • (다른 어드민 계정으로 로그인한 다음 삭제 가능)

보안 권장사항

  • 비밀번호는 다른 서비스와 다르게 설정하세요.
  • 최소 8자 이상, 영문 대소문자 + 숫자 조합 권장.
  • 사용하지 않는 계정은 즉시 삭제하세요.
  • 운영자 변경 시(퇴사 등) 즉시 비밀번호 변경 또는 계정 삭제.

자주 묻는 질문

Q. 비밀번호를 잊어버렸어요.

  • 다른 어드민 계정으로 로그인 후 → 운영자 계정 페이지에서 비밀번호 변경 가능
  • 모든 어드민 계정의 비밀번호를 잊은 경우 개발자에게 문의 (Firestore에서 직접 재설정 필요)

Q. admin 계정의 이름을 바꿀 수 있나요?

  • 현재는 이름 변경 기능이 없습니다. 향후 추가될 수 있습니다.

Q. 한 명에 여러 권한을 줄 수 있나요?

  • 현재는 최고 관리자 / 운영자 두 단계만 있습니다.
  • 모든 운영자는 모든 메뉴에 접근할 수 있으며, 단지 admin 계정만 삭제 보호됩니다.

Q. 운영자가 동시에 여러 명 로그인할 수 있나요?

  • 네, 각자 다른 PC에서 동시에 로그인 가능합니다.

12. 약관·정책

메뉴: 시스템 → 설정 → 약관/정책

마이페이지의 개인정보 처리지침과 이용약관 본문을 직접 작성·수정합니다.

사용 흐름

  1. 메뉴 진입 → 자동으로 기본 템플릿이 화면에 채워져 있습니다.
  2. 상단 탭에서 개인정보 처리지침 또는 이용약관 선택
  3. 제목과 본문 수정
  4. 저장 버튼 → 사용자 화면에 즉시 반영

화면 구성

상단 탭

  • 개인정보 처리지침 — 마이페이지 → 개인정보 처리지침에 노출
  • 이용약관 — 마이페이지 → 이용약관에 노출

탭 전환 시 저장하지 않은 변경사항이 있으면 확인 메시지가 뜹니다.

입력 영역

  • 제목 — 페이지 상단에 보일 제목 (기본값: 개인정보처리방침 / 서비스 이용약관)
  • 본문 — 실제 약관/정책 내용

본문 작성 규칙

  • 줄바꿈은 그대로 사용자 화면에 보입니다.
  • 빈 줄을 넣으면 문단이 분리됩니다.
  • 별도 마크다운/HTML 없이 텍스트로만 작성하셔도 됩니다.

메타 정보

  • 마지막 수정 일시 — 가장 최근 저장 시각
  • 수정자 — 어떤 어드민 계정이 수정했는지

버튼

버튼동작
기본 템플릿화면을 기본 본문으로 되돌림 (저장 안 함)
되돌리기마지막에 저장된 상태로 복귀
저장Firestore에 저장 + 사용자 화면 즉시 반영

사용자 화면 노출

저장 즉시 반영되며, 다음 위치에서 사용자가 볼 수 있습니다.

  • 마이페이지 → 개인정보 처리지침 메뉴
  • 마이페이지 → 이용약관 메뉴
  • 회원가입 시 약관 동의 페이지

자동 시드

이 페이지에 처음 들어가면 기본 본문이 자동으로 등록되어 사용자 화면에 노출됩니다.

이후로는 어드민이 수정한 내용으로만 노출됩니다.

⚠️ 두 탭 모두 한 번씩 들어가야 두 문서가 다 등록됩니다.

(한 탭만 들어가면 다른 문서는 빈 상태가 됩니다)

자주 묻는 질문

Q. 저장 안 한 상태로 다른 메뉴로 이동했어요.

  • 변경사항은 사라집니다. 다시 작성해 주세요.
  • 다음 버전부터 페이지 이탈 시 경고를 추가할 수 있습니다.

Q. 사용자가 변경된 약관에 다시 동의해야 하나요?

  • 현재는 별도 동의 절차 없이 바로 노출됩니다.
  • 중대한 약관 변경 시 별도 공지로 안내하시는 것을 권장합니다.

Q. 이전 버전 약관을 보존할 수 있나요?

  • 현재는 최신 버전만 저장됩니다.
  • 이전 버전 보존 기능이 필요하시면 개발자에게 문의해 주세요.

Q. 약관에 외부 링크나 이미지를 넣을 수 있나요?

  • 현재는 텍스트만 지원됩니다.
  • HTML/마크다운 지원이 필요하시면 개발자에게 문의해 주세요.
