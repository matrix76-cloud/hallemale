# 2026-05-06 작업 정리

배포 끝: https://halle-bf789.web.app

## 1. 다크모드 전면 적용 (78개 파일)

- **인프라**: 이미 깔려있음 — `ThemeContext`(기본 dark), `lightTheme`/`darkTheme` (`src/theme/theme.js`), App.js의 `ThemedApp`
- **마이페이지 → 화면 모드 토글** 동작 정상 (5/6 commit `5cc9f75` 시점부터)
- 5명 에이전트 병렬로 폴더별 분배 (auth/common, my, team+player, matching, community/chat/legal/admin/etc)
- 직접 작업: LoginPage, 홈 카드 6개(TeamProfileSection/PlayerRanking/TeamRanking/FavoriteTeams/FavoritePlayers/WinningTeams), MatchingManagePage, AiRecommendedTeamsSection, CommunityListPage, NotificationsPage

**Why:** 행렬이 형이 처음 "다크모드 제대로 적용 안 됨" → 점진적으로 "모두 바꿔야 함" 명시. 톤 가이드: 카드는 `theme.colors.card`, 페이지는 `bg`, 텍스트는 `textStrong/Normal/Weak` 위계, 결과/시그널 색은 다크에서도 가독 OK라 보존.

**How to apply:** 새 컴포넌트 추가할 땐 모든 색을 `theme.colors.*` 토큰으로. mode 분기 패턴: `theme.mode === "dark" ? darkValue : lightValue`. keyframes 안 색은 CSS 변수로 빼서 분기. 인라인 `style={{}}` 색상은 styled prop variant 또는 useTheme 훅으로.

## 4. 신고/차단 시스템 (회원/팀)

- `src/services/adminTeamBlockService.js` — blockTeam, unblockTeam, listBlockedTeams

- `src/services/teamReportService.js` — createTeamReport, listTeamReports, updateTeamReportStatus

- `AdminTeamsReportsPage` (신규) / `AdminTeamsBlocksPage` (신규)
- 신고 페이지: pending/resolved/rejected/all 탭 + [차단][기각] 액션
- 차단 페이지: [해제] 액션

### 사용자측
- `PlayerProfilePage` 미디어 섹션 끝에 "🚩 부정 사용자 신고하기" 링크 + 모달 (isSelf 가드)
- `TeamProfilePage` 끝에 "🚩 부정 팀 신고하기" 링크 + 모달 (자기 팀 아닌 경우만)
- `playerService.getPlayerProfile` / `teamService.getTeamProfile` 반환 객체에 `blocked/blockedReason/blockedAt/blockedBy` 필드 추가 (안 추가하면 가드 안 뜸 — 핵심 함정)

### 클라이언트 가드 (공통 컴포넌트)
- `src/components/common/BlockedOverlay.jsx` — 반투명 backdrop blur + 사유 + 차단일 + [뒤로가기][새로고침]
- `src/components/common/BlockedAuthGate.jsx` — App 루트 통합. 로그인된 사용자 `userDoc.blocked === true` 감지 시 BlockedOverlay 표시. **signOut 호출 안 함** (그 사이 어드민이 풀어주면 새로고침으로 즉시 정상 진입). 어드민 영역 제외.
- `PlayerProfilePage` — 다른 사람이 차단된 사용자 프로필 진입 시 BlockedOverlay
- `TeamProfilePage` — 누구든 차단된 팀 진입 시 BlockedOverlay
- `AuthContext` — 차단 시 `window.alert` 제거 (BlockedOverlay 통일)
- BlockedOverlay onBack: `nav(-1)` + history 비었으면 `/home` fallback

**Why:** 행렬이 형이 "alert 말고 커스텀 다이얼로그", "그 사이에 해제할 수 있으니" (signOut 하지 마라), "다른 사람이 차단된 사용자 들어가면 팀과 같이 나오게".

## 5. 광고 관리 (홈 배너 + 이벤트 팝업)

### 메뉴
- 광고 관리 (하위 2개)
  - 홈 배너 → `/admin/banners`
  - 이벤트 팝업 → `/admin/popups`

### Firestore
- `banners/{id}` — title, desc, imageUrl, storagePath, side, textAlign, order, active
- `event_popups/{id}` — title, body, imageUrl, storagePath, linkUrl, linkLabel, order, active, startAt, endAt

### 서비스
- `src/services/bannersService.js`
- `src/services/eventPopupsService.js`
- 이미지 업로드는 `mediaService.uploadCompressedImageMedia` 재사용 (1080px 자동 압축)

### 어드민 페이지
- `AdminBannersPage` — 홈 배너 등록/수정/삭제, 1200×600 (2:1) 권장
- `AdminEventPopupsPage` + `AdminEventPopupsSection` — 이벤트 팝업, 720×900 (4:5) 권장, 시작/종료 일시 (비우면 항상)

### 사용자측
- `HomeHeroBanner` 동적 로드 — Firestore active 배너 우선, 없으면 코드 안 BANNERS 4개 fallback
- `EventPopupModal` (App.js 통합) — 페이지 진입 시 active+기간내+오늘안버린 팝업 1개 모달 표시
  - **[오늘 그만보기]** — `localStorage.popup.dismissed.{id}` 에 오늘 날짜 저장
  - **[이벤트 바로 가기]** — `linkUrl` 있으면 그곳, 없으면 `/event/{id}` 페이지로
  - 어드민 영역 / 로그인 전 페이지(/welcome,/login,/signup,/find-,/link-phone) 비활성
- `EventPage` `/event/:id` — 팝업 상세. 자체 헤더 없음 (MainLayout 헤더 사용)

**Why:** 행렬이 형이 "광고 관리 메뉴 + 하위 두 개 (홈 배너/이벤트 팝업)". 등록된 이벤트 1건 (Firestore에 시드 완료됨, 시드 코드는 제거됨).

## 미해결 / 메모

- **firestore.rules 보안 규칙 — 여전히 전면 허용 ⚠️**: 클라이언트가 차단된 사용자라도 데이터 read는 가능. 진짜 차단은 서버 룰에서 막아야 함. (별도 작업 필요)
- **Storage 보안 규칙 — 이미지 업로드 동작 중**: rule이 풀려있는 상태일 가능성. 확인 필요.
- 다크모드 적용 시 인라인 색상 대부분 styled-component prop으로 옮김. 일부 admin 페이지의 select 인라인 style은 OS 기본 UI라 그대로 둠 (다크 모드 무관 어드민 라이트 강제됨).
