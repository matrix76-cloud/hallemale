- `src/components/home/WinningTeamsSection.jsx` (필요 시 표시만)

📝 **작업 내용**:
1. **현황**: `WinningTeamsSection`은 실데이터(clubs)지만 "순위 Top5"일 뿐 연승 필터 없음. `WinningTickerBar`는 `teamsMock`의 `streak` 하드코딩 사용.
2. `homeService.js`의 `normalizeClubDoc()`(83~114줄)에서 `stats.recentResults`를 함께 노출.
3. 연승 계산 헬퍼 추가(기존 로직 참고: `src/components/team/TeamStatsSection.jsx:63-88` `computeStreakFromRecentResultsLast`). `recentResults`(배열 끝이 최신)에서 **마지막부터 같은 결과 연속 개수** 계산 → `{ type:"WIN"|"LOSE", count }`.
4. `homeService.js`에서 `winningTeamsHighlight` 생성 시 **연승(count≥2, type=WIN)인 팀만 필터 + streakLabel("N연승")** 부여. (연승팀 없으면 빈 배열 → 섹션 자연스럽게 숨김/대체)
5. `WinningTickerBar.jsx`: `teamsMock` import 제거, **homeData context의 winningTeamsHighlight**(또는 동일 서비스 데이터) 사용. 연승팀 없으면 티커 미표시.
6. `WinningTeamsSection.jsx:182`은 이미 `t.streakLabel` 처리 대비됨 — homeService가 streakLabel을 주도록만 하면 됨.

✅ **완료 조건**: 홈에서 연승팀이 **실제 clubs.stats.recentResults 기반 연속 승리**로 표시되고, mock import 제거. 연승팀이 없으면 깨지지 않고 자연 숨김. 빌드 통과.
